Express框架

Express是Node.js的后端框架。原生Node.js会有很多的问题，比如：
（1）呈递静态页面很不方便，需要处理每个HTTP协议，还要考虑304问题。
（2）路由处理代码不清晰直观，需要很多正则表达式和字符串函数。
（3）不能集中精力写业务，要考虑很多其他的东西。

中间件

中间件（Middleware） 是一个函数，它可以访问请求对象（request object (req)）, 响应对象（response object (res)）, 和 web 应用中处于请求-响应循环流程中的中间件，一般被命名为 next 的变量。

中间件的功能包括：

执行任何代码。
修改请求和响应对象。
终结请求-响应循环。
调用堆栈中的下一个中间件。
路由在有冲突的情况下，只会匹配到第一个路由，匹配到了就不会继续向下匹配了。如果想要路由继续向下匹配，需要在回调函数中增加next参数，并调用next()。

express render

res.render手里有response对象，可以把视图响应给客户端。

express获取GET请求传来的参数

在原声node中，处理GET请求需要引用url模块来识别参数字符串，但在Express中，直接提供了这样的功能，可以直接使用req.query来获取.

express获取POST请求传来的参数

POST请求在express中不能直接获得，必须使用body-parse模块。使用后，将可以用req.body得到参数。但是如果表单中含有文件上传，那么还是需要使用formidable模块。
